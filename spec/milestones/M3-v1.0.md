## M3-12：生产构建 + .exe 打包

### 目标

`npm run build` 一键输出可部署产物，支持便携分发和 `.exe` 打包。

### 构建命令

```bash
npm run build        # 前端打包 (dist/client/)
npm run build:dist   # 便携分发 (dist/stuwise/)
npm start            # 生产启动
```

### 输出结构

```
dist/
  client/              ← Vite 前端
  stuwise/
    server.mjs          ← esbuild ESM bundle (供 Bun compile)
    client/             ← 前端静态文件
    tools/ skills/      ← 运行时目录
    start.bat start.sh  ← 启动脚本
    stuwise.exe         ← Bun 编译 (需 Windows + Bun)
```

### .exe 打包

使用 **Bun** 编译（nexe/pkg 不兼容 Node 24）：
```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
npm run build:dist
cd dist\stuwise
bun build server.mjs --compile --outfile stuwise.exe
```

Bun 将运行时嵌入 exe，目标机器无需安装 Node.js。API Key 不预置，用户启动后在 ⚙ 设置。

### 已知 Bug 记录

| 问题 | 解决方案 |
|------|---------|
| pi-agent-core `loadSkills` 使用 `ignores` 库，传绝对路径给相对路径接口。Linux 不报错，Bun 编译 exe 后严格检查，抛 `RangeError` | 自写 `server/skills-loader.ts`（纯 Node.js `fs`，不走 ignores） |
| nexe 无 Node 24 预编译二进制，`--build` 需 VS C++ 工具链 | 改用 Bun compile |
| pkg 只支持到 Node 18 | 改用 Bun compile |
| Bun compile 将 `.gitkeep` 也打包，路径转换有问题 | `build-dist.ts` 跳过 `.gitkeep` 复制 |
| Vue emit 多参数：`$event` 只捕获第一个参数，`...$event` 把字符串展开成字符 | 所有 emit 链改用显式箭头函数 `(a,b) => emit(...)` |
| Skill toggle：`<label>` + `<input>` 浏览器原生 toggle 与 `:checked` 冲突 | 改为纯 `<div>` + `.on` class |
