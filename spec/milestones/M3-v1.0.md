## M3-2：修复 B3 — Session 膨胀 + compaction

### 问题

多次对话后 session JSONL 超过模型 context window，导致 `stopReason: "error"`。

### 方案

利用 `harness.compact()` API 在 session 过大时自动压缩。

### 变更文件

- **修改** `server/harness.ts` — 启动时检查 session 大小，必要时 compact
- **新增** `server/routes/compact.ts` — 手动触发 compact 端点

### 实现概要

```typescript
// 启动时检查
const tokenEstimate = await estimateTokens(session);
if (tokenEstimate > THRESHOLD) {
  await harness.compact();
}

// 手动触发
app.post("/api/compact", async (c) => {
  const result = await harness.compact();
  return c.json(result);
});
```

### 验收条件

- [ ] 多次对话后不再出现 context overflow error
- [ ] compaction 后对话连贯性不被破坏

## M3-3：Session 管理（列表/切换/新建）

### 范围

用户可以在前端管理对话 Session（列出/切换/新建）。

**同时优化持久化方案**：前端不再用 localStorage 缓存消息，改为从后端 API 加载对话历史，后端 JSONL 为唯一数据源。

### 变更文件

- **新增** `server/routes/session.ts` — CRUD 端点
- **修改** `server/harness.ts` — 支持多 session + 暴露消息列表
- **新增** `client/src/components/SessionList.vue` — 侧边栏
- **修改** `client/src/App.vue` — 集成 SessionList
- **修改** `client/src/composables/useAgent.ts` — 移除 localStorage，从 API 加载历史

### API

```
GET    /api/session          → 列出所有 session
POST   /api/session          → 新建 session（返回 sessionId）
DELETE /api/session/:id      → 删除 session（含 JSONL 文件）
PUT    /api/session/:id      → 切换当前 session
GET    /api/session/:id      → 获取 session 对话历史
```

### 持久化优化

| | Before（当前） | After（v1.0） |
|---|---|---|
| 数据源 | 后端 JSONL + 前端 localStorage | 后端 JSONL 唯一数据源 |
| 加载 | 页面刷新从 localStorage 恢复 | 切换 session 时 `GET /api/session/:id` 加载 |
| 写入 | `useAgent` 调用 `saveMessages()` | SSE 事件实时追加，无 localStorage 写 |
| 一致性 | 可能不一致 | 前后端同步 |

`useAgent.ts` 移除 `loadMessages()` / `saveMessages()` 函数，改为异步加载。

### 验收条件

- [ ] 左侧显示 Session 列表
- [ ] 点击可切换 Session，对话内容独立
- [ ] 可新建/删除 Session
- [ ] 切换 Session 后对话历史从后端加载
- [ ] localStorage 不再存储消息

### 实现概要

```typescript
// harness 加载时支持过滤器
const allSkills = await loadSkills(env, "./skills");
const enabledSkills = allSkills.filter(s => enabledSet.has(s.name));
const skillBlock = formatSkillsForSystemPrompt(enabledSkills);

// harness 维护启用状态
let skillState: Map<string, boolean> = new Map();
function setSkillEnabled(name: string, enabled: boolean) {

  // 重新注入 System Prompt
  await harness.setResources({ skills: getEnabledSkills() });
}
```

## M3-6：配置管理（API Key + 模型切换）

### 范围

### 变更文件

- **新增** `client/src/components/ConfigPanel.vue` — 配置面板
- **新增** `server/routes/config.ts` — 配置读写端点
- **修改** `server/harness.ts` — 支持运行时切换 model
- **修改** `client/src/App.vue` — 集成 ConfigPanel

### API

```
GET    /api/config     → 获取当前配置（provider, modelId, hasApiKey）
PUT    /api/config     → 更新配置（apiKey, provider, modelId）
```

### 实现概要

```typescript
// server/harness.ts
let currentModel: Model<any>;
let currentApiKey: string;

async function updateConfig(config: { apiKey?: string; provider?: string; modelId?: string }) {
  if (config.apiKey) currentApiKey = config.apiKey;
  if (config.provider && config.modelId) {
    currentModel = getModel(config.provider, config.modelId);
    await harness.setModel(currentModel);
  }
}

function getConfig() {
  return {
    provider: currentModel.provider,
    modelId: currentModel.id,
    hasApiKey: !!currentApiKey,
  };
}
```

### 验收条件

- [ ] 前端可输入 API Key，不暴露完整 Key（只显示 hasApiKey）
- [ ] 前端可切换 Provider/Model，切换后对话正常
