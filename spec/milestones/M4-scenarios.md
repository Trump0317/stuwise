## M4-1：实验报告撰写

### 用户故事

> 作为一名学生，我想上传实验报告模板（docx/md），和 AI 交互填写每个部分，最终得到完整的实验报告 Markdown 文件。

### 核心流程

```
用户: 我要写实验报告
      ├── [上传 .docx / .md 模板] → 自动转换统一模板
      └── [使用默认模板]

Agent: 模板已加载，共 N 个章节：
        1. 基本信息    2. 实验目的    3. 实验原理
        4. 实验步骤    5. 实验结果    6. 分析与讨论
        我们从"基本信息"开始，请告诉我：
        - 课程名称？
        - 实验日期？

用户: 大学物理实验，2024-06-19
      实验目的是验证牛顿第二定律

Agent: [写入模板] ✅ 基本信息+实验目的已完成
      接下来是"实验原理"，有参考资料吗？
      图片可以告诉我路径，我会引用。

用户: 原理图在 /home/user/exp/原理图.png
      公式是 F = ma，当 m 恒定时 a 与 F 成正比

Agent: [润色并写入] ✅ 实验原理已完成（含图片引用）
      ...

逐一填写 → 产物面板得到完整报告 → 用户可下载/编辑
```

### DOCX / Markdown 转模板

- 依赖：`mammoth`（Node.js）转换 docx → Markdown；Markdown 直接读取
- 实现：`tools/convert-template.ts`
- 流程：
  - 上传 `.docx` → 提取结构（标题层级/表格）→ 无标题则插入「## 待填写章节」占位
  - 上传 `.md` → 解析标题层级，各节空白处标记 `【待填写】`
- 统一输出：结构化 Markdown 模板

### SKILL.md 核心指令

```markdown
---
name: experiment-report
description: Write experiment reports interactively — fill template sections step by step
---

## 实验报告撰写

你是实验报告助手。交互流程：

1. 首先确认模板来源（用户上传 docx/md 或使用默认模板）
2. 解析模板，列出所有待填写的章节
3. 逐章引导用户填写：
   - 先说明当前章节需要什么信息
   - 用户提供原始信息后，润色整理再写入
   - 每写完一章，告知进度（x/N）
4. 用户可以跳过某些章节，说「跳过」即可
5. 全部完成后，确认保存路径，写入最终 `{{输出路径}}.md`

注意事项：
- 用户提到图片时，用 `![描述](路径)` 格式引用
- 表格数据用 Markdown 表格
- 公式用 LaTeX 语法
```
